This document provides a summary of the key test scenarios executed
for the To-Do List web application.  
Detailed scenario execution and tracking were performed using Trello.

---

## CREATE (Add To-Do)
- Verify user can add a new to-do using the input field
- Verify to-do is added when Enter key is pressed
- Verify empty to-do is not added
- Verify item count increases when a new to-do is added

---

## READ (View To-Do)
- Verify all to-do items are displayed correctly
- Verify active to-do items are shown under Active filter
- Verify completed to-do items are shown under Completed filter
- Verify item count displays correct number of active items

---

## UPDATE (Modify To-Do)
- Verify user can mark a to-do as completed using checkbox
- Verify completed to-do shows visual change
- Verify completed to-do can be marked active again
- Verify item count updates when to-do status changes

---

## DELETE (Remove To-Do)
- Verify completed to-do items can be cleared
- Verify Clear action removes only completed items
- Verify active items are not affected by Clear action
- Verify item count updates after deletion

---

## Note
Test scenarios were executed and managed in Trello.
This document provides a high-level view of the testing coverage.
